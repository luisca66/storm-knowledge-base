# Modulo: Chats de IA

> **Estado:** desarrollado con evidencia de clases reales (grupo martes mayo 2026 — Carmen, Mario, Montse, Karla; y otros alumnos individuales).

---

## Objetivo

Que el alumno sepa elegir el modelo correcto segun su plan y su tarea, entienda las modalidades disponibles (thinking, deep research, web search) y use el chat como cerebro estratégico — no como reemplazo del agente.

---

## Distincion central

**Chat = cerebro estratégico. Agente = ejecutor.**

El chat no ejecuta tareas en el mundo real. Conversa, planea, estructura, analiza y genera texto. Sus limites como agente son parte de lo que lo hace util como planificador. Entender esto desde el inicio evita frustración.

## Modalidades del chat

### Thinking (razonamiento extendido)
- El modelo piensa antes de responder.
- Util para problemas complejos, planificacion de proyectos, decisiones con muchas variables.
- Tarda mas. No es para preguntas rapidas.

### Deep Research (investigacion profunda)
- El modelo busca, sintetiza y cita fuentes reales.
- Util para investigar un tema, comparar opciones, entender un campo desconocido.
- El alumno debe revisar las fuentes citadas — no asumir que son correctas.

### Web Search (busqueda en tiempo real)
- El modelo accede a informacion actual.
- Util para precios, noticias, disponibilidad, datos recientes.
- No reemplaza investigacion profunda.

### Conversacion estandar
- Sin modos especiales. Rapida, util para la mayoria de las tareas cotidianas.

---

## Superpoderes del chat para trabajar con agentes

El chat no ejecuta — pero es el mejor planificador:

1. **Entrevistador:** extrae toda la informacion del proyecto antes de pasarsela al agente.
2. **Generador de instrucciones:** produce el `.md` que el agente va a ejecutar.
3. **Verificador:** puede auditar el output del agente en un segundo chat.
4. **Generador de criterio:** ayuda al alumno a saber que esperar del agente.

Ver: `01_metodo/flujo_chat_md_agente.md` y `leccion_06`.

## Filosofia silenciosa

El chat es poderoso exactamente por sus limites: no actua, no guarda, no ejecuta sin confirmacion. Eso lo hace un entorno seguro para pensar. El alumno que entiende esto usa el chat para diseñar — y al agente para construir.

## Ejercicio practico

El alumno elige una pregunta real de su trabajo y la pasa por dos modalidades distintas. Luego:
- Que diferencia encontro?
- Que modalidad usaria para que tipo de tarea?
- Cual de las dos respuestas revisaria mas cuidadosamente?

## Errores comunes

- Usar thinking para preguntas simples (innecesario y lento).
- Asumir que deep research siempre cita fuentes verificadas.
- Confundir el modelo con la herramienta (ChatGPT no es lo mismo que GPT-5).
- Usar el chat para tareas que necesitan un agente.
- Subir informacion sensible sin pensar en privacidad.

---
