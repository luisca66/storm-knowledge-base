# Reglas para diseño de lecciones

Este documento traduce los patrones de alumnos en reglas practicas para construir el curso base.

## Regla 1: cada leccion debe producir una accion observable

Una leccion no esta completa si solo explica. Debe dejar una accion:

- prompt escrito
- decision tomada
- fuente organizada
- app creada
- salida evaluada
- bitacora completada
- ruta elegida

## Regla 2: empezar con una victoria rapida

La primera experiencia debe demostrar que IA puede hacer algo util o sorprendente. Despues se introduce criterio.

## Regla 3: no abrir con filosofia abstracta

La filosofia es el compañero silencioso. Primero se vive el problema; despues se nombra el concepto.

Ejemplo:

- Primero: "ChatGPT dijo esto, lo creemos?"
- Despues: "Esto se llama problema de conocimiento o epistemologia."

## Regla 4: usar la secuencia central

Toda leccion debe reforzar:

1. Pensar primero.
2. Usar IA segundo.
3. Evaluar despues.

## Regla 5: enseñar temprano el mapa de herramientas

El alumno debe distinguir:

- chat para pensar y escribir
- Deep Research para investigar
- NotebookLM para estudiar fuentes
- agentes para ejecutar tareas
- Codex/Claude Code para proyectos, archivos o codigo
- herramientas visuales para diseño, imagen o video

## Regla 6: privacidad antes de permisos

Antes de conectar o subir informacion:

- identificar datos sensibles
- anonimizar si hace falta
- revisar permisos
- explicar que no todo debe entrar a la IA

## Regla 7: adaptar por perfil sin perder estructura

La estructura es la guia; Luis decide el camino vivo.

| Perfil | Adaptacion |
|---|---|
| Niño/adolescente | microvictorias, juego, retos cortos |
| Profesional creativo | utilidad, privacidad, entregables reales |
| Explorador avanzado | mapa de herramientas, cierres breves |
| Adulto mayor | ritmo, significado, proyectos elegidos |
| Pragmatico impaciente | sesiones cortas, checklist de rescate |

## Regla 8: cerrar siempre

Cada clase debe cerrar con:

- logro principal
- dificultad
- tarea o siguiente paso
- herramienta usada
- que debe registrar Luis

## Regla 9: medir avance por autonomia

El curso no se mide solo por herramientas vistas. Se mide por cuanto puede hacer el alumno sin Luis:

- Nivel 0: observa
- Nivel 1: sigue instrucciones
- Nivel 2: usa con guia
- Nivel 3: propone y ejecuta
- Nivel 4: crea proyectos
- Nivel 5: elige estrategia y evalua riesgos

## Regla 10: cada leccion debe alimentar el sistema

Despues de probar una leccion, Luis o un agente debe revisar:

- que funciono
- que perfil respondio mejor
- donde hubo friccion
- que ejercicio debe cambiar
- si la leccion debe dividirse o moverse
