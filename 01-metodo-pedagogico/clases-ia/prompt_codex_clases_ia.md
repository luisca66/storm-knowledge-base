## 1. Contexto del proyecto

Luis está desarrollando un sistema profesional para planear, registrar, mejorar y dar seguimiento a sus clases personalizadas de inteligencia artificial.

Actualmente Luis ya cobra estas clases y quiere volverlas más profesionales. No quiere un curso rígido tradicional; quiere un sistema vivo, adaptativo y consultable, que le ayude a preparar clases, documentar lo que ocurre en cada sesión, registrar el progreso de cada alumno, generar ideas para próximas clases y mantener una memoria permanente del trabajo.

Las clases son mixtas: presenciales y en línea.

Duración habitual:
- 2 horas por clase.

Precio objetivo:
- 2000 MXN por clase por persona.

El proyecto no debe estar conectado con Storm Studios Learning. Debe ser una línea separada.

Luis se identifica como “vibe coder”: no es un programador tradicional, pero usa herramientas de IA, agentes y sistemas modernos para resolver problemas reales, crear proyectos, experimentar, aprender y enseñar desde la experiencia directa de quien no empezó sabiendo código.

Esta condición es una ventaja pedagógica central del proyecto: Luis entiende lo que se siente no saber nada técnico y puede acompañar a alumnos que también parten desde niveles bajos o medios.

## 2. Propósito central

El propósito de estas clases no es solamente enseñar herramientas de IA.

El propósito es enseñar a las personas a usar IA para aprender, resolver, crear, organizarse y volverse más autónomas.

La promesa pedagógica central es:

> El alumno debe aprender a usar la IA de tal manera que, después de avanzar con Luis, pueda seguir aprendiendo nuevas habilidades sin depender de Luis ni de nadie más.

La clase debe potenciar:
- Creatividad.
- Productividad.
- Organización.
- Capacidad de aprendizaje autónomo.
- Pensamiento crítico.
- Uso práctico de agentes.
- Capacidad para transformar problemas personales, académicos o profesionales en proyectos concretos asistidos por IA.

No se debe prometer:
- Que Luis desarrolla sistemas empresariales complejos.
- Que Luis implementa infraestructura profesional para empresas grandes.
- Que se sustituyen expertos técnicos cuando un proyecto ya requiere ingeniería formal, seguridad, bases de datos complejas, arquitectura empresarial o cumplimiento legal.

Sí se puede prometer:
- Acompañamiento práctico.
- Aprendizaje adaptado.
- Mejora de productividad y creatividad.
- Organización de proyectos.
- Uso inteligente de IA.
- Desarrollo de criterio para trabajar con herramientas modernas.
- Introducción gradual a agentes, automatización ligera, Codex, Antigravity, Cowork, Claude Code, GitHub y flujos de trabajo asistidos.

## 4. Filosofía pedagógica

Crea documentos que expliquen esta filosofía:

1. Las clases son adaptativas.
2. Cada alumno es diferente.
3. El punto de partida suele ser bajo o medio, pero no se debe subestimar al alumno.
4. El aprendizaje se activa cuando el alumno descubre que puede construir cosas reales con IA.
5. La clase debe partir de problemas reales del alumno siempre que sea posible.
6. Luis debe poder improvisar, pero el sistema debe registrar y ordenar esa improvisación.
7. La meta no es que el alumno memorice funciones, sino que aprenda un modo de pensar y trabajar con IA.
8. El pensamiento crítico es central.
9. La IA no debe presentarse como magia, sino como una herramienta poderosa con límites, riesgos y responsabilidades.
10. La experiencia de Luis como vibe coder debe formularse como una ventaja metodológica: aprender haciendo, experimentar, resolver sin esperar a dominar toda la teoría técnica, pero sabiendo cuándo un proyecto requiere mayor nivel profesional.
