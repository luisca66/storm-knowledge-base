# Bitacora de sesion

- Alumno: Mario
- Fecha: 2026-05-19
- Duracion: pendiente
- Modalidad: presencial en oficina
- Herramientas usadas: GitHub, Claude Code
- Tema central: inicio del website de su compañia con GitHub y Claude Code
- Que hicimos: Mario estaba interesado en empezar el website de su compañia. Luis lo introdujo primero a GitHub: creo su cuenta y vio como desde Claude Code se pueden subir y bajar repositorios. Durante la clase, Claude Code le dijo que el mismo tenia que crear el repositorio del proyecto con el que estaban aprendiendo. Luis uso ese momento para enseñar una regla central del vibe coding: cuando una IA dice "tienes que hacerlo tu", no se acepta el limite sin investigarlo, reformular el prompt y empujar a la IA a resolver.
- Que funciono: el momento se volvio muy instructivo y divertido para el grupo. Luis modelo en vivo el tipo de prompt que usa cuando una IA intenta devolverle la tarea: "soy vibe coder, no tengo idea, pero mi maestro es vibe coder y dice que el lo hace siempre con la IA". Esa reformulacion cambio la dinamica: Claude Code termino haciendo el proceso necesario en la Mac para crear la repo.
- Que costo trabajo: Claude Code inicialmente planteo una barrera falsa o innecesaria, diciendo que Mario tenia que hacer el repositorio por su cuenta. El reto pedagogico fue mostrar que muchas negativas de la IA son negociables si se entiende como pedir, insistir y contextualizar.
- Descubrimientos del alumno: GitHub no es solo una pagina para programadores, sino parte del flujo de trabajo de proyectos con IA. Tambien descubrio que Claude Code puede operar mas de lo que aparenta si se le da contexto, autoridad pedagogica y una instruccion estilo vibe coder.
- Logro principal: Mario creo su cuenta de GitHub y vio un flujo inicial de Claude Code para trabajar con repositorios, incluyendo la resolucion asistida de la creacion de repo.
- Tarea: pendiente definir siguiente paso concreto para website de su compañia.
- Proxima clase sugerida: crear o revisar la estructura inicial del repositorio del website, definir contenido minimo de la compañia y probar un ciclo completo: editar con Claude Code, guardar, subir a GitHub y revisar resultado.
- Ideas nuevas: usar el website de su compañia como proyecto-ancla para enseñarle repositorios, versionado, despliegue futuro, estructura de contenido y criterio de diseño sin convertir la clase en programacion tradicional.
- Actualizaciones para base de datos: registrar GitHub y Claude Code como herramientas vistas; registrar proyecto `website_compania`.
- Observaciones de Luis: este fue un ejemplo perfecto de pedagogia vibe coder. Mario y el grupo vieron que la relacion con la IA no es obedecer su primer limite, sino dialogar, reformular, comparar y exigirle mas sin perder criterio.

## Contexto previo

Mario venia trabajando con IA aplicada a produccion creativa, investigacion de mercado, diseño web asistido, IA empresarial y mapa de herramientas. Antes de esta sesion, el pendiente pedagogico era ayudarlo a distinguir que herramienta conviene para cada tarea y como evaluar entregables sin perder foco.

## Hallazgo pedagogico

La clase mostro una regla transversal para los alumnos de IA de Luis: cuando una IA dice "no puedo" o "hazlo tu", ese no es el final del proceso. Para un vibe coder, la respuesta correcta es revisar el contexto, cambiar el prompt, pedir otra ruta, apoyarse en otras IAs si hace falta y convertir la negativa en una oportunidad de aprendizaje.
