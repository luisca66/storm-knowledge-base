# github_profesional

## Enfoque

GitHub profesional: commits claros, ramas, pull requests, issues, README y proteccion de datos.

## Conceptos que Luis debe dominar

- Vocabulario basico.
- Casos simples que puede enseñar.
- Riesgos y limites.
- Señales de que se necesita un especialista.
- Ejercicios pequeños para alumnos.

## Practica recomendada

Crear un ejemplo propio, documentarlo, probarlo con datos ficticios y registrar que funciono y que no.

## Limite profesional

No presentar este tema como implementacion empresarial avanzada hasta tener mas practica, pruebas y criterio tecnico.
