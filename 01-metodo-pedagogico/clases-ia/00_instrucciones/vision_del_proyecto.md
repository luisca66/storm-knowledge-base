# Vision del proyecto

Este documento es la brujula del proyecto. Si un agente, asistente o Luis tiene duda sobre hacia donde debe crecer el sistema, debe leer primero esta vision.

## Que quiere lograr Luis con este sistema

El sistema tiene dos metas principales:

1. Documentar todas las clases personalizadas de IA.
2. Crear un curso base con lecciones secuenciales y progresivas que sirvan como guia para Luis al dar clase.

El repositorio no existe solo para guardar notas. Debe ayudar a recordar, ordenar, mejorar y convertir la experiencia de clase en un plan de estudios cada vez mas claro.

## Tipo de clases

Las clases deben poder adaptarse a un rango amplio de alumnos:

- Niños y niñas en escuela primaria.
- Adolescentes.
- Adultos con baja o media confianza tecnologica.
- Profesionales activos economicamente.
- Personas de distintas areas que quieren usar IA para aprender, trabajar, crear, organizarse o resolver problemas concretos.

El curso base debe ser suficientemente ordenado para guiar a Luis, pero suficientemente flexible para adaptarse a perfiles muy distintos.

## Que no debe volverse el proyecto

El proyecto no debe convertirse en una base de datos esteril.

No basta con acumular expedientes, CSVs o bitacoras. El sistema debe poder:

- Detectar tendencias.
- Encontrar patrones entre alumnos.
- Sugerir mejoras.
- Ayudar a pulir lecciones.
- Mostrar que ejercicios funcionan mejor.
- Identificar donde los alumnos se atoran.
- Proponer rutas nuevas segun edad, perfil, interes y nivel.

Si el repositorio no ayuda a pensar mejor las clases, esta fallando.

## Papel de la IA

La IA es el centro operativo del curso. Las clases enseñan a usar IA para aprender, crear, organizarse, investigar, producir, automatizar y construir proyectos.

La filosofia es el compañero silencioso del sistema. No compite con la IA ni desplaza el foco principal, pero debe estar presente como base de pensamiento critico, logica, criterio humano y responsabilidad.

La fusion correcta es:

- Enfasis visible: IA practica.
- Base silenciosa: filosofia, pensamiento critico y pensamiento logico.

El alumno debe aprender herramientas, pero tambien debe aprender a pensar mejor con ellas.

## Papel de Luis como vibe coder y profesor

Luis es el guia y el Sensei.

El curso base es una guia, no una jaula. Todo pasa por la capacidad de Luis para improvisar, leer al alumno, adaptar el ritmo, detectar energia, elegir el ejemplo correcto y convertir una necesidad especifica en una experiencia de aprendizaje.

Luis no enseña desde la rigidez de un curso tradicional. Enseña desde experiencia directa, criterio, sensibilidad, experimentacion y capacidad de resolver con IA aunque no venga de una ruta tecnica tradicional.

La estructura debe apoyar esa improvisacion, no apagarla.

## Que significa plan de estudios

Para este proyecto, un plan de estudios significa lecciones bien identificadas, progresivas y reutilizables.

Cada leccion debe tener:

- Nombre claro.
- Objetivo.
- Nivel sugerido.
- Conceptos clave.
- Herramientas.
- Actividad guiada.
- Ejercicio practico.
- Señal de avance.
- Errores comunes.
- Variantes por perfil.
- Siguiente leccion sugerida.

El plan debe permitir que Luis entre a una clase y sepa que no esta olvidando nada importante, aunque despues adapte el camino en vivo.

## Prioridad inmediata

Antes del martes 19 de mayo de 2026 deben estar listas por lo menos las primeras cuatro lecciones del curso base.

La prioridad maxima es la primera leccion.

La primera leccion debe funcionar incluso cuando Luis no sabe nada de la alumna o alumno antes de empezar. Debe servir como diagnostico, introduccion, victoria rapida y apertura del metodo.

## Como debe ayudar un agente en este proyecto

Cada vez que un agente como Codex, Gemini, Claude Code o Cowork trabaje aqui, debe ayudar a Luis a convertir experiencia en sistema.

Debe revisar:

- Si las bitacoras de clases dadas estan completas.
- Si los perfiles de alumnos estan actualizados.
- Si los CSVs reflejan lo que paso en clase.
- Si hay tendencias entre alumnos.
- Si hay ejercicios que funcionan mejor que otros.
- Si una leccion necesita pulirse.
- Si una ruta necesita simplificarse o hacerse mas ambiciosa.

El agente debe sugerir caminos para seguir mejorando cada leccion, sin borrar la sensibilidad ni la improvisacion de Luis.

## Regla central

El proyecto debe producir memoria viva, no archivo muerto.

Cada clase debe alimentar el sistema. Cada bitacora debe mejorar el curso. Cada alumno debe enseñar algo sobre como enseñar mejor IA.
